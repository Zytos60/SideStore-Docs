---
title: 3-App 限制繞過
description: 教學如何使用 SparseRestore 漏洞通過 SideStore 安裝超過 3 個應用程式。
---

:::note

SparseRestore 的「繞過 3 個應用程式限制」漏洞 **僅適用於 iOS/iPadOS 版本 17.0 至 18.1 beta 4（不包括 17.7.1）**。如果您不在這些版本中且無法使用該漏洞，**請檢查版本是否正確，或者參考替代方案**。

:::

## 必備條件
* 一台運行 SparseRestore 漏洞支持版本（參見上述注意事項）的 iDevice。
* 至少有一個空閒的應用程式安裝槽位。
* SideStore 版本 0.5.9 或更高版本。您可以在[這裡](https://github.com/SideStore/SideStore/releases)找到最新版本。
* 執行此漏洞時，您**必須**關閉 iDevice 上的 **Find My**（尋找我的裝置），否則將無法使用。以下是關閉 Find My 的步驟：
  - 打開「設定」應用程式。
  - 點選頂部的您的名字。
  - 點選「尋找我的」。
  - 點選「尋找我的 iPhone/iPad」。
  - 關閉「尋找我的 iPhone/iPad」。
  - 您可能需要輸入您的 Apple ID 密碼以確認。
  
  > **注意：** 如果您無法關閉 Find My，可能是由於啟用了「被盜裝置保護」。請在嘗試關閉 Find My 之前先關閉「被盜裝置保護」。

## 步驟
1. 安裝 SparseBox。有多種方法可以完成此操作，但對於本教學，請使用 SideStore 安裝。您可以在[這裡](https://nightly.link/khanhduytran0/SparseBox/actions)找到最新版本。

2. 在 SideStore 中勾選「禁用應用程式限制」，確保 SideStore **已關閉**（未在背景執行），然後打開 SparseBox。

3. 啟用 StosVPN，選擇您的配對檔案並點擊「繞過 3-app 限制」。這應該會顯示一個日誌輸出。等待它顯示 `idevicebackup2 exited with code -18` 或 `idevicebackup2 exited with code -25`。

4. 重新打開 SideStore，然後在 SideStore 設定中向下滾動。點擊「清除緩存」，這將允許 SideStore 檢測到該漏洞。

5. 測試該漏洞是否成功（根據您的空閒應用槽位數量，您可能需要安裝多於一個應用以進行測試）。如果出現有關限制的錯誤，請重新打開 SideStore 或重試該漏洞。

## 注意事項
由於某些限制，您每次額外安裝 3 個應用程式時都需要重新執行此漏洞。

如果您需要將 **Find My** 用於其他用途，現在可以重新打開它。然而，如果您需要安裝或刷新更多應用程式，則需要再次關閉它。

## 替代方案
如果漏洞對您無效，或者您的 iDevice 處於不支持的版本，您可以使用其他方法來同時使用超過 3 個應用程式。LiveContainer 由 Duy Tran Khanh 開發，能夠有效解決這個問題。請參考相關文件獲取更多資訊。
